USER MANUAL

This user manual describes how to use the Comparison Tool (http://www.letshpc.org/main) of the Let’s HPC Platform.

The Comparison tool has plotting features which allow the users to analyze and compare the data from various benchmark experiments that have been contributed by the members of the HPC Community. This tool is extremely useful when the users want to analyze, compare or learn from the performance that others have achieved using various implementation approaches on various machines and programming environments.

Step 1: Open the comparison tool webpage ( http://www.letshpc.org/main ). This will open the following screen

Step 2: Select a category from the options that have been provided in the Category dropdown.

Step 3: Once the category has been selected, you will need to select a problem from the dropdown provided.

Step 4 : Select the Architecture for which you want to compare the results. (Note : As of now, we have data only for the Shared Memory architecture, but we have members who are actively working to add benchmarked data for the Distributed Memory and the Heterogenous architectures.)

Step 5: Click on the Get Machine, Approach and Programming Environments button

We now have three options available. Compare Machines, Approaches or Programming Environments. Step 6 - Step 12 vary on the basis of the the option that you chose.

Comparing Machines 

Step 6 : Click on the Compare Machines Tab to activate the relevant options.

Step 7: Select the machines that you want to compare from the dropdown. Please note that multiple options can be selected in this dropdown menu. The machines which have a blue background have been selected for comparison. If you want to unselect a machine, click the X button next to the machine name.

Step 8: Select the approach on the basis of which you want to compare the machines. The options in the Approach dropdown menu are based on the Problem that was selected in Step 3.

Step 9: Select the Programming Environment. The options in this dropdown are based on the architecture which was selected in Step 4.

Step 10 : Click on the Set Data Button.

Step 11: All the machines that you selected will appear in a separate row.

Step 12 : For each machine, select the threads that you want to plot. And then click on the Update button. The dropdown is a multiple selection dropdown similar to Step 7. For the basic analysis, you can also select if you want to plot the data based on the End-to-End timings of the program or only the Algorithmic Part.

Step 13 : Selecting the Statistic. You can select the statistic (mean, median, range, standard deviation) to plot from the dropdown which is located below the plot.

Step 14 : THe metric can be selected by clicking on the metric selector button. A new pop-up would be created, which will have all the metrics listed.

Step 15 : The graph can be customized to your specific needs by clicking on the Customize Graph button. This will allow you to change the X and Y axis in the log scale, and set custom limits for the X and the Y Axis.

Compare Approaches

Step 6 : Click on the Compare Approaches Tab to activate the relevant options.

Step 7: Select the approaches that you want to compare from the dropdown. Please note that multiple options can be selected in this dropdown menu. The approaches which have a blue background have been selected for comparison. If you want to unselect an approach, click the X button next to the approach name.

Step 8: Select the machine on which you want to compare the approaches.

Step 9: Select the Programming Environment. The options in this dropdown are based on the architecture which was selected in Step 4.

Step 10 : Click on the Set Data Button.

Step 11: All the approaches that you selected to compare will appear in a separate row.

Step 12 : For each approach, select the threads that you want to plot. And then click on the Update button. The dropdown is a multiple selection dropdown similar to Step 7. For the basic analysis, you can also select if you want to plot the data based on the End-to-End timings of the program or only the time taken to run the algorithm.

Step 13 : Selecting the Statistic. You can select the statistic (mean, median, range, standard deviation) to plot from the dropdown which is located below the plot.

Step 14 : THe metric can be selected by clicking on the metric selector button. A new pop-up would be created, which will have all the metrics listed.

Step 15 : The graph can be customized to your specific needs by clicking on the Customize Graph button. This will allow you to change the X and Y axis in the log scale, and set custom limits for the X and the Y Axis.
